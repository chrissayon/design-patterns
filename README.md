# Design Patterns

Design patterns are reusable solutions to the same recurring problems that come in software design.

## What are we solving for?

More often than not in programming, the same common structural or architectural problems will show appear. Of course this changes based on the requirements that you're dealing with.

Thanks to the experience of developers more wiser than I, they have created recognisable and usable approaches and patterns to deal with these issues.

## Should I be using design patterns at work?

That is up to you.

Just because you've learnt a new design pattern, don't treat it like the golden hammer to use for every solution. Look at the problem and decide whether it's suitable.
